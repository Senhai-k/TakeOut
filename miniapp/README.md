# miniapp

用户端微信小程序，使用原生小程序开发。当前界面为中文版本，布局使用 `rpx` 适配通用手机机型。

## 当前页面

- `pages/index/`：首页/附近商家入口
- `pages/search/`：搜索商家/菜品
- `pages/merchant-menu/`：商家菜单/点餐页
- `pages/cart/`：购物车
- `pages/dish-detail/`：菜品详情
- `pages/order-confirm/`：确认订单
- `pages/order-list/`：订单列表
- `pages/order-detail/`：订单详情
- `pages/address-list/`：地址列表
- `pages/address-edit/`：新增/编辑地址
- `pages/profile/`：我的

## 当前能力

- 中文界面
- 橙色年轻化外卖 UI 风格
- 通用手机机型适配，不绑定某个固定屏幕尺寸
- 4 个底部导航：首页、购物车、订单、我的
- 搜索作为首页搜索框进入的二级页面
- 首页展示定位、搜索、圆形分类入口和附近商家
- 搜索页支持热门搜索、分类筛选、商家/菜品结果
- 商家菜单页支持分类切换、菜品列表、一键加购、底部购物车浮条
- 菜品详情规格选择、辣度选择、备注和加入购物车
- 购物车按商家隔离，切换商家时提示清空旧购物车
- 购物车按规格/辣度/备注区分商品行
- 商品删除和购物车汇总
- 购物车明细、优惠码输入、费用汇总
- 确认订单
- 地址管理优先同步后端，后端不可用时使用本地存储
- 提交订单前会把本地购物车同步到后端购物车，再创建订单
- 订单列表和订单跟踪详情优先读取后端，后端不可用时使用本地存储
- 优先请求本地后端，后端不可用时使用本地 mock 数据

## 本地调试

1. 使用微信开发者工具导入 `miniapp` 目录。
2. 使用测试 AppID。
3. 在微信开发者工具中开启：

```text
详情 -> 本地设置 -> 不校验合法域名、web-view、TLS版本以及HTTPS证书
```

4. 启动后端：

```bash
cd server
mvn spring-boot:run
```

5. 小程序默认请求地址：

```text
http://127.0.0.1:8080/api
```

配置位置：

```text
app.js
```

如果使用真机调试，需要把 `127.0.0.1` 改成电脑局域网 IP。
