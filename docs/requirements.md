# 外卖小程序需求文档

## 1. 项目概述

本项目是一个面向餐饮商家的外卖小程序系统，主要提供用户在线点餐、购物车、下单、支付、订单跟踪，以及商家商品管理、订单处理和平台后台管理等功能。

第一版系统以“单商家/多商家可扩展、自配送”为目标，优先完成外卖业务主流程，不实现复杂骑手调度、实时轨迹、复杂营销和平台分账。

## 2. 项目目标

- 用户可以通过微信小程序完成浏览商品、加入购物车、提交订单和查看订单状态。
- 商家可以通过后台管理商品、分类、库存和订单。
- 管理员可以维护用户、商家、商品、订单和系统基础数据。
- 系统具备后续扩展多商家、优惠券、评价、配送和数据统计的能力。

## 3. 用户角色

### 3.1 普通用户

使用微信小程序点餐的消费者。

主要能力：

- 登录小程序
- 浏览商品
- 管理收货地址
- 加入购物车
- 提交订单
- 支付订单
- 查看订单状态
- 取消订单
- 评价订单

### 3.2 商家

负责店铺运营和订单处理的人员。

主要能力：

- 管理商品分类
- 管理商品信息
- 设置商品上下架
- 查看订单
- 接单或拒单
- 修改订单状态
- 查看营业数据

### 3.3 平台管理员

负责系统整体维护的人员。

主要能力：

- 管理用户
- 管理商家
- 管理商品
- 管理订单
- 查看统计数据
- 维护系统配置

## 4. 功能需求

### 4.1 用户端小程序

#### 4.1.1 登录

- 用户可以使用微信授权登录。
- 首次登录时系统自动创建用户账号。
- 用户可绑定手机号。

#### 4.1.2 首页

- 展示店铺信息，包括店铺名称、公告、营业状态、起送价、配送费。
- 展示商品分类。
- 展示推荐商品或热销商品。
- 当店铺休息时，用户可以浏览商品，但不能提交订单。

#### 4.1.3 商品浏览

- 用户可以按分类查看商品。
- 商品信息包括名称、图片、价格、销量、库存、简介。
- 已下架商品不在用户端展示。
- 库存为 0 的商品显示售罄，不能加入购物车。

#### 4.1.4 购物车

- 用户可以将商品加入购物车。
- 用户可以修改商品数量。
- 用户可以删除购物车商品。
- 购物车应展示商品总数、商品总价、配送费和应付金额。
- 商品下架或库存不足时，提交订单前应提示用户。

#### 4.1.5 收货地址

- 用户可以新增、编辑、删除收货地址。
- 用户可以设置默认地址。
- 地址信息包括收货人、手机号、详细地址、门牌号。

#### 4.1.6 下单

- 用户选择购物车商品和收货地址后可以提交订单。
- 系统生成待支付订单。
- 下单时需要校验店铺营业状态、商品上下架状态、库存和起送金额。
- 订单应包含商品明细、收货地址、商品金额、配送费、实付金额和备注。

#### 4.1.7 支付

- 用户可以使用微信支付。
- 支付成功后，订单状态变为“已支付待接单”。
- 支付失败或取消支付时，订单保持“待支付”。
- 超过指定时间未支付的订单自动取消。

#### 4.1.8 订单

- 用户可以查看订单列表。
- 用户可以查看订单详情。
- 用户可以查看订单状态变化。
- 待支付订单可以取消。
- 已支付但商家未接单的订单可以申请取消。
- 已完成订单可以评价。

#### 4.1.9 评价

- 用户可以对已完成订单进行评价。
- 评价内容包括评分和文字内容。
- 一个订单只能评价一次。

### 4.2 商家端

#### 4.2.1 商家登录

- 商家使用账号密码登录后台。
- 登录后根据权限展示功能菜单。

#### 4.2.2 店铺管理

- 商家可以维护店铺名称、公告、营业状态、起送价和配送费。
- 商家可以设置店铺营业或休息。

#### 4.2.3 分类管理

- 商家可以新增商品分类。
- 商家可以编辑分类名称和排序。
- 商家可以启用或禁用分类。
- 分类下存在商品时，不建议直接删除分类。

#### 4.2.4 商品管理

- 商家可以新增商品。
- 商家可以编辑商品名称、图片、价格、库存、分类、简介。
- 商家可以设置商品上架或下架。
- 商家可以删除未产生订单关联的商品。
- 已产生订单的商品建议逻辑删除或下架。

#### 4.2.5 订单管理

- 商家可以查看新订单。
- 商家可以接单。
- 商家可以拒单，并填写拒单原因。
- 商家可以将订单状态更新为制作中、配送中、已完成。
- 商家可以查看订单详情，包括商品明细、收货地址、备注和支付金额。

#### 4.2.6 数据统计

- 商家可以查看今日订单数。
- 商家可以查看今日销售额。
- 商家可以查看近 7 日订单趋势。
- 商家可以查看热销商品排行。

### 4.3 平台管理端

#### 4.3.1 管理员登录

- 管理员使用账号密码登录后台。
- 系统应校验账号状态。

#### 4.3.2 用户管理

- 管理员可以查看用户列表。
- 管理员可以查看用户详情。
- 管理员可以禁用或启用用户。

#### 4.3.3 商家管理

- 管理员可以新增商家。
- 管理员可以编辑商家信息。
- 管理员可以启用或禁用商家。

#### 4.3.4 订单管理

- 管理员可以查看全部订单。
- 管理员可以按订单号、用户、商家、订单状态、时间筛选订单。
- 管理员可以查看订单详情。
- 管理员可以处理异常订单。

#### 4.3.5 商品管理

- 管理员可以查看全部商品。
- 管理员可以下架违规商品。

#### 4.3.6 系统配置

- 管理员可以维护基础配置，如订单自动取消时间、配送费默认值等。

## 5. 订单状态

订单状态定义如下：

| 状态 | 说明 |
| --- | --- |
| 待支付 | 用户已提交订单，尚未完成支付 |
| 已支付待接单 | 用户已支付，等待商家处理 |
| 商家已接单 | 商家确认订单 |
| 制作中 | 商家正在制作商品 |
| 配送中 | 商品已交付配送 |
| 已完成 | 订单完成 |
| 已取消 | 订单取消 |
| 退款中 | 已支付订单正在退款 |
| 已退款 | 退款完成 |

状态流转规则：

```text
待支付 -> 已支付待接单 -> 商家已接单 -> 制作中 -> 配送中 -> 已完成
待支付 -> 已取消
已支付待接单 -> 已取消/退款中 -> 已退款
```

## 6. 核心业务流程

### 6.1 用户下单流程

```text
用户登录
-> 浏览商品
-> 加入购物车
-> 选择收货地址
-> 提交订单
-> 创建待支付订单
-> 微信支付
-> 支付成功
-> 商家接单
-> 制作/配送
-> 订单完成
```

### 6.2 商家接单流程

```text
商家登录后台
-> 查看新订单
-> 接单
-> 设置制作中
-> 设置配送中
-> 设置已完成
```

### 6.3 订单取消流程

```text
待支付订单
-> 用户取消
-> 订单关闭

已支付待接单订单
-> 用户申请取消
-> 系统退款
-> 订单关闭
```

## 7. 非功能需求

### 7.1 性能需求

- 商品列表接口响应时间建议小于 1 秒。
- 订单提交接口需要避免重复提交。
- 支付回调接口需要保证幂等处理。

### 7.2 安全需求

- 用户端接口需要校验登录状态。
- 后台接口需要校验管理员或商家权限。
- 密码需要加密存储。
- 支付回调需要验签。
- 用户只能查看自己的订单和地址。

### 7.3 可靠性需求

- 订单创建、库存扣减和支付状态更新需要保证数据一致性。
- 微信支付回调可能重复通知，系统必须幂等处理。
- 未支付订单需要定时自动关闭。

### 7.4 可维护性需求

- 后端按用户、商品、购物车、订单、支付、后台管理等模块划分。
- 接口返回格式保持统一。
- 异常码和错误信息需要统一维护。
- 数据库字段命名应清晰一致。

## 8. 第一版 MVP 范围

第一版必须完成：

- 用户微信登录
- 商品分类和商品展示
- 购物车
- 地址管理
- 提交订单
- 微信支付或模拟支付
- 订单状态管理
- 商家后台商品管理
- 商家后台订单管理
- 管理员登录

第一版暂不实现：

- 骑手抢单
- 实时配送轨迹
- 多城市运营
- 复杂优惠券系统
- 平台分账
- 秒杀活动
- 会员等级

## 9. 验收标准

- 用户可以从浏览商品到完成下单走通完整流程。
- 商家可以在后台看到订单并修改订单状态。
- 用户端订单状态可以随商家操作同步变化。
- 后台可以新增、编辑、上架、下架商品。
- 系统能防止重复支付回调导致订单状态异常。
- 基础页面和接口无明显报错。
